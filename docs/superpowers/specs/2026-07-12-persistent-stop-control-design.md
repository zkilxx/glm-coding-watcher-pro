# 持久停止控制与弹窗按钮布局设计

## 目标

解决 GLM 页面自动刷新后 Chrome 扩展弹窗关闭、用户难以及时停止监测的问题，并修正弹窗底部操作按钮未对齐的问题。

## 交互设计

- 监测运行时，在 GLM 页面右下角显示一个固定悬浮控制条。
- 控制条显示绿色运行状态、“正在监测”文字和醒目的“停止”按钮。
- 页面刷新后，内容脚本从后台读取运行状态并重新创建控制条。
- 点击网页中的“停止”后，向后台发送 `STOP_MONITORING`，停止检测与刷新，并立即移除控制条。
- 未运行、已经成功点击或用户停止后，不显示控制条。
- 控制条使用 Shadow DOM 隔离样式，避免受 GLM 页面 CSS 影响，也不修改页面业务内容。
- 扩展弹窗中的“开始监测”和“停止”改为等宽两列，统一高度、圆角和对齐方式；窄内容下文字不换行。

## 数据流

1. 弹窗启动监测，后台保存标签页运行状态并通知内容脚本。
2. 内容脚本进入运行状态时创建悬浮控制条。
3. 页面刷新后内容脚本重新注入，通过 `GET_STATE` 恢复运行状态和控制条。
4. 用户点击悬浮“停止”，内容脚本发送包含当前标签页来源的 `STOP_MONITORING`。
5. 后台将运行状态设为停止并回传；内容脚本清理定时器、观察器和控制条。

## 错误处理

- 停止消息失败时，按钮恢复可点击并显示“重试停止”，避免假装已经停止。
- 重复点击停止按钮不会创建多个请求。
- 控制条使用高层级但保持较小尺寸，不遮挡订阅卡片区域。
- 不处理验证码、登录、限流、风控、订单确认或支付。

## 测试

- 自动测试验证运行状态会创建控制条，停止状态会移除控制条。
- 自动测试验证网页停止按钮发送 `STOP_MONITORING`。
- 自动测试验证刷新恢复路径仍调用 `GET_STATE` 并重建控制条。
- 样式测试验证弹窗操作区为等宽两列，按钮尺寸一致。
- 在 Chrome 中验证：启动监测、页面刷新、控制条重新出现、网页按钮停止、弹窗按钮对齐。
