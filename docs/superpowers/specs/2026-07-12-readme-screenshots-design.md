# README 功能截图设计

## 目标

在 GitHub README 中加入真实、可长期访问的功能截图，让用户无需安装即可理解扩展的主要操作流程。

## 截图内容

1. 插件主界面总览：标题、状态、监测模式、套餐与运行信息。
2. 套餐选择与优先级：月付、季付、年付筛选，多选目标和上下调整顺序。
3. 监测设置：DOM 检测间隔、声音提醒、桌面通知、等宽开始与停止按钮。
4. 网页状态卡：右上角运行状态、刷新次数、检测间隔和停止按钮。

## 文件与排版

- 图片保存到 `docs/images/`，使用稳定英文文件名。
- README 新增“功能截图”章节，每张图前有小标题，图后有一段简短中文说明。
- 图片使用仓库相对路径，保证 GitHub README 和 Download ZIP 均可访问。
- 截图裁掉无关浏览器区域；不展示账号头像、验证码内容或其他个人信息。
- 不伪造购买成功、支付或验证码通过状态。

## 验证

- 检查所有 README 图片路径对应真实文件。
- 检查图片可解码且尺寸适合 GitHub 页面。
- 检查 README Markdown 渲染结构和替代文本。
- 在 GitHub 仓库页面确认图片正常显示。
