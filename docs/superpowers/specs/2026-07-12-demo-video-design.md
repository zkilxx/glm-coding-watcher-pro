# GLM Coding 监测助手演示视频设计

## 交付规格

- 时长：约 60 秒
- 画幅：16:9
- 分辨率：1920×1080
- 格式：MP4（H.264）
- 语言：简体中文字幕，无配音
- 声音：轻量背景音乐，音量不干扰字幕阅读
- 用途：GitHub README、B站、群聊和产品演示

## 视觉风格

- 延续插件的蓝白配色、圆角卡片和清晰信息层级。
- 使用真实插件截图和界面组件，不展示账号头像、验证码细节或支付信息。
- 通过平滑缩放、局部框选、光标移动和短字幕突出操作重点。
- 不伪造购买成功；“自动点击”使用流程示意说明，并明确后续验证与支付由用户手动完成。

## 分镜

### 0–5 秒：标题

- 标题：`GLM Coding 订阅监测助手 Pro`
- 副标题：`多套餐优先级 · 自动刷新 · 单次点击`

### 5–15 秒：套餐选择

- 展示插件总览。
- 聚焦月付、季付、年付筛选。
- 字幕：`自动读取不同订阅周期`

### 15–25 秒：抢购优先级

- 勾选两个目标套餐。
- 展示上移、下移和移除控件。
- 字幕：`同时选择多个目标，从上到下依次尝试`

### 25–35 秒：监测设置

- 聚焦 DOM 检测间隔、声音提醒、桌面通知。
- 字幕：`设置检测频率与本地提醒`

### 35–48 秒：运行状态

- 点击开始监测。
- 展示网页右上角状态卡，刷新次数递增。
- 字幕：`目标不可购买时刷新页面，并记录真实刷新次数`

### 48–55 秒：自动点击逻辑

- 使用界面示意：购买按钮变为可用，插件自动点击一次并停止刷新。
- 字幕：`按钮可用时只点击一次，防止重复操作`

### 55–60 秒：安全边界与链接

- 字幕：`验证码、登录、订单确认与支付仍需手动完成`
- 显示 GitHub：`github.com/zkilxx/glm-coding-watcher-pro`

## 验证标准

- 成片分辨率为 1920×1080，时长 55–65 秒。
- 所有字幕均在安全区域内且无裁切。
- 音频无爆音，背景音乐不过度响亮。
- GitHub 地址清晰可读。
- 不包含个人信息、验证码细节、支付信息或误导性成功承诺。
- 抽取关键帧进行视觉检查，并完整播放验证 MP4 可解码。
